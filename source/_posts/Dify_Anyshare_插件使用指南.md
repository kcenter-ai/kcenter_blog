## 概述

Anyshare Dify 插件是一款用于从 Anyshare 文档库中召回文档切片的工具插件。通过该插件，您可以在 Dify 工作流中集成 Anyshare 的文档检索能力，实现基于企业文档知识的 RAG（检索增强生成）应用。

### 主要功能

- 从 Anyshare 文档库中检索相关文档切片
- 支持切片检索
- 可与企业现有文档权限体系集成
- 支持在工作流中灵活配置召回参数

## 前置准备

在使用 Anyshare Dify 插件之前，您需要提前准备好以下信息：

### 1. 应用账户凭证

| 参数 | 说明 | 获取方式 |
|------|------|----------|
| 应用账户 ID | 用于身份认证 | 联系 Anyshare 管理员申请 |
| 应用账户密钥 | 用于身份认证 | 联系 Anyshare 管理员申请 |

### 2. 文档库权限

确保为应用账户分配了对应的文档库权限，且该文档库满足以下条件：

- 已开启全文解析功能
- 已开启向量切片功能

### 3. 服务访问地址

准备 Anyshare 服务的访问 base_url，例如：

```
https://your-anyshare-server:8080
```

或生产环境地址：

```
https://your-anyshare-domain.com
```

## 插件下载与安装

### 插件包

插件包文件：`anyshare_retrieval.difypkg`

插件包下载：[anyshare_retrieval.difypkg](/assets/anyshare_retrieval.difypkg)

### 安装步骤（本地文件方式）

1. 登录 Dify 平台

2. 进入 **插件管理** 页面
   - 点击 Dify 平台右上角的「插件」按钮
   - 进入插件管理页面

3. 选择 **上传本地文件** 方式

4. 上传 `anyshare_retrieval.difypkg` 文件

5. 等待插件安装完成

> ⚠️ **重要**：在某些 Dify 配置下，可能需要关闭第三方插件验证才能正常使用。

## 工作流配置

### 创建工作流

1. 在 Dify 平台中创建新的应用或进入现有应用

2. 选择 **工作流** 类型

3. 点击「创建空白工作流」

4. 在工作流画布中，创建类似以下的工作流：

![](/assets/17755702749519.jpg)

### 工作流节点说明

典型的工作流应包含以下节点：

| 节点类型 | 功能说明 |
|----------|----------|
| 开始节点 | 接收用户输入的查询问题 |
| Anyshare 召回节点 | 从 Anyshare 文档库检索相关切片 |
| 输出节点 | 用于查看召回节点的输出结果 |

---

## 召回节点参数配置

召回节点是整个插件的核心，用于配置从 Anyshare 检索文档切片的参数。

### 参数配置界面

![](/assets/17755703873895.jpg)

### 主要参数说明

| 参数名称 | 必填 | 说明 | 示例值 |
|----------|------|------|--------|
| 查询语句 | 是 | 用户问题 | `anyshare是什么？` |
| 基础 URL | 是 | Anyshare 服务访问地址 | `https://192.168.1.100:8080` |
| username | 是 | 应用账户 ID | `your_app_id` |
| password | 是 | 应用账户密钥 | `your_app_key` |
| 限制 | 是 | 召回的切片大小限制 | `10` |
| 输出类型 | 是 | 召回的切片类型 | `doc` |

完成以上配置后，需要对整个工作流进行测试。

## 执行结果查看

### 查看检索结果

测试完成后，可以查看各节点的执行结果。

### 结果查看界面

![](/assets/17755708559592.jpg)

### 检查项

| 检查项 | 说明 |
|--------|------|
| 召回节点输出 | 确认检索到了相关文档切片 |
| 文档相关性 | 检查召回的文档是否与查询相关 |
| 输出生成结果 | 检查输出的结果是否成功输出 |

### 常见结果状态

- **成功**：工作流正常运行，召回相关文档
- **无结果**：未召回任何文档，可能需要调整参数
- **错误**：节点执行出错，检查配置是否正确

## 常见问题

### Q1: 插件安装失败怎么办？

**解决方案**：
1. 确认 `.difypkg` 文件完整未损坏
2. 关闭第三方插件验证
3. 检查 Dify 版本是否支持该插件

### Q2: 召回结果为空？

**可能原因**：
1. 应用账户没有文档库权限
2. 文档库未开启全文解析或向量切片


**解决方案**：
1. 联系管理员确认权限配置
2. 确认文档库已正确配置

### Q3: 召回结果不准确？

**解决方案**：
1. 调整 limit 参数，增加召回数量
2. 优化查询问题的表述

### Q4: 如何确认文档库已开启全文解析和向量切片？

**检查方式**：
1. 登录 Anyshare 管理控制台
2. 进入智能搜索-文档解析策略
3. 确认对应文档库或文件目录的「全文解析」和「向量切片」功能已启用

### Q5: 应用账户 ID 和密钥如何获取？

**获取方式**：
联系 Anyshare 管理员申请，需要提供：
- 应用名称
- 需要的文档库权限范围
- 预期用途说明
